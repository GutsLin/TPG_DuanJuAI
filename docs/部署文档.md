## 2. 日常部署(推荐):一键脚本

本地仓库根目录执行:

```bash
bash scripts/deploy.sh
```

一条命令完成:打包 → 上传 → 服务器备份 → 覆盖代码 → 构建镜像并重启 → 健康检查。

### 首次使用

脚本交互式生成 `scripts/deploy.config.local.sh`(服务器 IP/用户名/密码/项目目录),该文件已 gitignore 且权限 600,**不会提交**。也可手动复制模板填写:

```bash
cp scripts/deploy.config.example.sh scripts/deploy.config.local.sh
chmod 600 scripts/deploy.config.local.sh
# 编辑填入真实值
```

### 参数

| 参数 | 作用 |
|---|---|
| (无) | 完整部署:同步代码 + 构建重启 + 健康检查 |
| `--dry-run` | 只打包/传输/同步代码,不构建不重启(用于预检) |
| `--with-infra` | 同时覆盖服务器的 `Dockerfile` 与 `docker-compose.yml`(**慎用**,见 §4) |
| `-y` | 跳过"工作区有未提交改动"等确认 |
| `-h` | 帮助 |

### 脚本行为细节

- **打包排除**:`.git`、`data/`、`node_modules`、`.env`、`.smoke-infra/`、`scripts/deploy.config.local.sh` 等,包体约 730K
- **默认保留服务器定制文件**:`Dockerfile`、`docker-compose.yml` 不同步(除非 `--with-infra`)
- **备份**:每次同步前把服务器当前代码打包到 `~/deploy-backups/code-<时间戳>.tar.gz`(排除 `data/`),自动清理只留最近 5 份
- **凭据安全**:密码仅写入临时 askpass 文件(本地 mktemp + 服务器 `~/.deploy-askpass.sh`),脚本退出时双端删除
- **验证**:构建后循环健康检查最多 90 秒,失败时给出日志查看命令

## 4. 服务器定制文件(重要,勿覆盖)

服务器版与仓库 master 存在**刻意差异**,一键脚本默认保留:

### Dockerfile(服务器版特有)

- 全部 3 个构建阶段配置 `npm config set registry https://registry.npmmirror.com`(国内构建必需)
- apt 源替换为 `mirrors.tencentyun.com`
- `COPY configs/config.example.yaml ./configs/config.yaml`
- 内置 `ENV AUTH_JWT_SECRET` 兜底值

### docker-compose.yml(服务器版特有)

- 挂载 `./configs:/app/configs:ro`
- `AUTH_JWT_SECRET=${AUTH_JWT_SECRET:-change-this-...}` 带回退(仓库版为 `:?` 强制要求)

如确需用仓库版覆盖,使用 `bash scripts/deploy.sh --with-infra`,并先确认仓库版已合入上述镜像源配置,否则国内构建会失败。

## 5. 数据库迁移

- 后端启动时自动执行 `backend/drizzle/` 中的迁移(`src/db/migrate.ts`,drizzle migrator 按 `meta/_journal.json` 顺序应用,已应用的记录在 `drizzle.__drizzle_migrations`)
- 迁移均为**增量 DDL**(加表/加列),对生产数据安全;`postgres` 容器与 named volume 在日常部署中不重建
- ⚠️ **教训记录**(2026-07-23):服务器库的迁移记录曾被人手工预写(整点/未来时间戳),导致 0004 迁移被跳过、两列缺失,已手工补建。如升级后发现"列不存在"类错误,先核对:

```bash
sudo docker exec tpg_duanjuai-postgres-1 psql -U huobao -d huobao_drama \
  -c "select * from drizzle.__drizzle_migrations order by created_at;"
```

必要时手动执行对应 `backend/drizzle/000N_*.sql` 中的 SQL 修复。

## 7. 回滚

一键脚本每次部署前自动备份代码(不含 `data/`,媒体文件不受影响):

```bash
ssh ubuntu@101.33.231.171
ls -t ~/deploy-backups/                                  # 选一份备份
tar -xzf ~/deploy-backups/code-<时间戳>.tar.gz -C ~/
cd ~/TPG_DuanJuAI && echo '<密码>' | sudo -S docker compose up -d --build
```

注意:回滚代码不会回滚数据库结构(迁移是单向增量);新增列/表对旧代码通常兼容。

## 8. 故障排查

| 症状 | 排查 |
|---|---|
| 构建失败(npm/apt 超时) | 确认用的是服务器定制 Dockerfile(镜像源);看 `~/TPG_DuanJuAI/build2.log` |
| 健康检查超时 | `sudo docker compose logs --tail=50 huobao-drama`;常见为 DB 未就绪或迁移报错 |
| 生成任务不执行 | Worker 是否在跑:`sudo docker ps \| grep worker`;日志:`sudo docker logs tpg_duanjuai-worker-1` |
| 权限 denied(docker) | ubuntu 不在 docker 组,一律 `sudo`;可选优化:`sudo usermod -aG docker ubuntu` 后重新登录 |
| 媒体文件 404 | 检查 `./data` bind mount:`sudo docker inspect tpg_duanjuai-huobao-drama-1 \| grep -A3 Mounts` |
| 磁盘空间 | `df -h /`;清理:`sudo docker system prune -f` |

## 9. 安全注意事项

- 服务器密码、`.env`、`scripts/deploy.config.local.sh` 均不进 git;`.gitignore` 已覆盖后两者
- `AUTH_JWT_SECRET` 生产必须保持长随机值(服务器 `.env` 已配置)
- 一键脚本通过 `SSH_ASKPASS`/临时文件传递密码,退出即删,不在进程列表/历史中留痕
- 建议后续加固:配置 SSH 密钥登录并禁用密码登录;`ubuntu` 加入 docker 组免 sudo
